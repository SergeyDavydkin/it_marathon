# it_marathon
Задача 1: «Круиз Робинсона»
Легенда
Корабль поврежден, но у нас есть аварийный шаттл, отчаянный агент Робинсон, готовый лететь за помощью, и мы как раз обнаружили планету, где мы сможем договориться о проведении спасательной экспедиции. Туда нам и надо отправиться. Проблема в том, что взлёт с нашей текущей планеты (назовем ее 
start_point
) требует чудовищных затрат топлива — настолько огромных, что каждый лишний килограмм на орбите стоит целого бака горючего.

Твоя задача – найти максимально оптимальный (по времени и топливу) путь до целевой планеты (назовем ее 
rescue_point
). P.s. запомни имена планет (начальной и целевой), они тебе пригодятся!

Что мы знаем:
на вывод единицы массы на орбиту нужно огромное количество топлива (считаем, что человек и баки с запасом кислорода массы не имеют);
на борту ограничен запас воздуха — полет не может длиться вечно;
после выхода на орбиту шаттл летит по автопрограмме:
половина имеющегося на борту топлива сразу тратится на разгон (с округлением до целых вниз);
при этом разгон принудительно прекращается, если при сжигании очередной единицы топлива будет пройдено больше половины от всего запланированного расстояния;
пока разгон не завершен, выполнять гравитационные маневры нельзя (но можно просто перемещаться по ребрам графа)
затем шаттл летит по инерции (равномерное движение без ускорений); при этом можно заложить в программу выполнение одного или нескольких гравитационных маневров, на которые будет тратиться время и топливо;
при приближении к цели оставшаяся часть топлива тратится на полное торможение шаттла, т.е. полет должен закончиться с нулевой скоростью
учитывай, что если мы взяли гравитационный разгон, то тормозить, соответственно, придется дольше; аналогично разгону, в процессе торможения никакие маневры выполнять нельзя;
работа двигателей на разгон и торможение осуществляется таким образом, что каждую секунду сжигается одна единица топлива:
учитывай, что чем меньше на борту топлива, тем меньше суммарная масса шаттла, следовательно, тем большее ускорение получит шаттл;
считай, что расход единицы топлива происходит мгновенно, после чего на одну секунду шаттл получает соответствующее его текущей (без учета только что потраченного топлива) массе ускорение;
на последней секунде торможения двигатели могут работать меньше секунды, чтобы не придать шаттлу скорость в обратном направлении; при этом все равно тратится полная единица топлива и ровно секунда на выполнение торможения (оставшееся время шаттл неподвижен).
Гравитационный маневр
Вместо прямого полета можно использовать систему космических тел (звезд, планет и лун) для гравитационных маневров. Каждый маневр дает прирост скорости, но требует точного расчета траектории и затрат топлива на коррекцию курса.

Система космических тел образует ориентированный граф, в котором узлы - это звезды и планеты, а ребра - допустимые прямые перелеты между телами. Движение возможно только по ребрам графа (нельзя произвольно менять курс).

Параметры каждого тела:

уникальный идентификатор;
масса тела (в условных единицах);
список допустимых радиусов маневра (в условных единицах).
Параметры каждого ребра:

идентификаторы начального и конечного тел;
расстояние между телами (метры).
Механика гравитационного маневра заключается в следующем:

при пролете вокруг космического тела шаттл разгоняется на определенную величину, зависящую от таких параметров как, например, радиус облета; соответственно, для каждого тела может быть спланировано несколько различных маневров, но выполнить при пролете мимо тела можно только один;
появляется дополнительный расход топлива на коррекцию траектории;
затрачивается дополнительное время на выполнение маневра.
Что нужно сделать в этой задаче
Написать код, который:

определит, возможно ли преодолеть найденный путь с имеющимися ресурсами (запасами топлива и воздуха);
найдет путь в графе космических тел между заданной парой звезд, при котором суммарное время полета будет минимально возможным в условиях ограниченности запасов топлива;
для каждого тела на пути выберет оптимальный гравитационный маневр. При этом допустимо пролететь мимо тела и не выполнять гравитационный маневр (простое перемещение по графу, затраты времени и топлива на корректировку курса к следующему телу в таком случае считать нулевыми).
API-интерфейс задачи
Все три задания в рамках Проектного этапа должны составлять ОДНО веб-приложение. Приложение должно запускать HTTP-сервер на порту 8000 и обрабатывать запрос (для задачи 1): 
POST /api/v1/robinson_cruise

Формат запроса (JSON):

{
  "mass_shuttle": 1000.0,
  "mass_fuel_unit": 1.0,
  "power_per_unit": 1000.0,
  "oxygen_time": 864000,
  "total_fuel": 7000,
  "fuel_consumption": 3.0,
  "bodies": [
    {
      "id": "planet1",
      "gravity_assists": [
        {
          "velocity_gain": 10000.0,
          "fuel_consumption": 40,
          "time_to_execute": 1095
        },
        {
          "velocity_gain": 24000.0,
          "fuel_consumption": 84,
          "time_to_execute": 2026
        }
      ]
    },
    {
      "id": "gas_giant",
      "gravity_assists": []
    }
  ],
  "edges": [
    {
      "from": "start_point",
      "to": "planet1",
      "distance": 2e8
    },
    {
      "from": "planet1",
      "to": "gas_giant",
      "distance": 1e5
    },
    {
      "from": "gas_giant",
      "to": "rescue_point",
      "distance": 4e6
    }
  ]
}

Формат ответа (JSON):

{
  "can_reach": true,
  "min_flight_time": 709698.8,
  "route": ["start_point", "planet1", "gas_giant", "rescue_point"]
}

Код ответа: 
200 OK

Формат ответа, если не можем долететь:

{
  "can_reach": false
}

Код ответа: 
200 OK

Формат ответа при вводе некорректного формата или диапазона значений (JSON):

{
  "status": "incorrect_input"
}

Код ответа: 
400 Bad Request

Где:

mass_shuttle
 — масса шаттла без топлива (кг);
mass_fuel_unit
 — масса единицы топлива (кг);
power_per_unit
 — ускорение, прилагаемое к единице массы за счет расхода одной единицы топлива (кг*м/с²);
oxygen_time
 — время, на которое хватит воздуха (секунды, целое число);
total_fuel
 — общий доступный запас топлива (целые ед.);
fuel_consumption
 — расход топлива – сколько единиц топлива сжигается за вывод единицы массы на орбиту при взлете;
bodies
 – космические тела, 0 <= len(bodies) <= 100:
id
 — уникальный идентификатор тела (строка);
gravity_assists
 — описание допустимых гравитационных маневров: для выполнения маневра потребуется 
time_to_execute
 секунд и 
fuel_consumption
 единиц топлива, а прирост скорости составит 
velocity_gain
 м/с;
конечные точки маршрута (start_point и rescue_point) в списке bodies не указываются, для них отсутствуют варианты грав. маневров;
edges
 – ребра:
from
 — идентификатор начального тела;
to
 — идентификатор конечного тела;
distance
 — расстояние между телами (метры);
в списке ребер должны хотя бы по одному разу упоминаться стартовая и конечная вершины;
can_reach
 — можем долететь или нет (булевое значение);
route
 — последовательность имен космических тел в выбранном пути, начиная со входной и заканчивая выходной;
min_flight_time
 — общее время полета (в секундах, одна цифра после запятой);
status
 — статус ответа приложения.
Учитывай, что важной частью решения задачи является проверка валидности входных данных. В числе возможных ошибок может быть как несоблюдение описанного выше формата (например, отсутствие или неправильное наименование части полей, ошибки в типах данных), так и некорректность значений (например, выход за допустимые диапазоны, нарушение связности данных или несоответствие условиям задачи).
Автоматическая проверка включает не только тесты на корректную обработку валидных запросов, но и тесты на некорректные входные данные.
Решение должно корректно обрабатывать такие случаи и возвращать ответ строго в указанном формате.
